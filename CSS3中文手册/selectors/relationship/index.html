选择符 css Selectors Module

关系选择符 Relationship Selectors

选择符 名称 版本 描述
E F 包含选择符(Descendant combinator) CSS1 选择所有被E元素包含的F元素。
E>F 子选择符(Child combinator) CSS2 选择所有作为E元素的子元素F。
E+F 相邻选择符(Adjacent sibling combinator) CSS2 选择紧贴在E元素之后F元素。
E~F 兄弟选择符(General sibling combinator) CSS3 选择E元素所有兄弟元素F。

E F 包含选择符

语法：

E F{srules}

说明：

与子选择符不同的是，包含选择符将会命中所有符合条件的后代，包括儿子，孙子，孙子的孙子...

兼容性：

  • 浅绿 = 支持
  • 红色 = 不支持
  • 粉色 = 部分支持
IE Firefox Chrome Safari Opera iOS Safari Android Browser Android Chrome
6.0+ 2.0+ 4.0+ 3.1+ 3.5+ 3.2+ 2.1+ 18.0+
事例： 
							.demo div{border:solid 1px #F00}
							
							<div class="demo">
								<div>0 
									<div> 1</div> 
									<div> 2</div>
									<div> 3</div> 
								</div>
							</div>

此例，如果使用.demo div，那么 0, 1, 2, 3 都有有边框；

E > F 子选择符

语法：

E > F{srules}

说明：

子选择符只能命中子元素，而不能命中孙辈。

兼容性：

  • 浅绿 = 支持
  • 红色 = 不支持
  • 粉色 = 部分支持
IE Firefox Chrome Safari Opera iOS Safari Android Browser Android Chrome
6.0 2.0+ 4.0+ 3.1+ 3.5+ 3.2+ 2.1+ 18.0+
7.0+
事例： 
							.demo > div{border:solid 1px #F00}

上面的html结构例子，如果使用 .demo > div，那么只有 0 有边框，即只有子元素会被命中；

E + F 相邻选择符

语法：

E + F { sRules }

说明：

选择紧贴在E元素之后F元素。 与兄弟选择符不同的是，相邻选择符只会命中符合条件的相邻的兄弟元素。

兼容性：

  • 浅绿 = 支持
  • 红色 = 不支持
  • 粉色 = 部分支持
IE Firefox Chrome Safari Opera iOS Safari Android Browser Android Chrome
6.0 2.0+ 4.0+ 3.1+ 3.5+ 3.2+ 2.1+ 18.0+
7.0+
事例： 
p + p{color:#f00;}
							<p>p1</p> 
							<p>p2</p> 
							<h3>这是一个标题</h3>
							<p>p3</p> 
							<h3>这是一个标题</h3>
							<p>p4</p>
							<p>p5</p>

此例，如果使用p + p{color:#f00;}，那么p2, p5将会变成红色；

E ~ F 兄弟选择符

语法：

E ~ F { sRules }

说明：

选择E元素后面的所有兄弟元素F。。与相邻选择符不同的是，兄弟选择符会命中所有符合条件的兄弟元素，而不强制是紧邻的元素。

兼容性：

  • 浅绿 = 支持
  • 红色 = 不支持
  • 粉色 = 部分支持
IE Firefox Chrome Safari Opera iOS Safari Android Browser Android Chrome
6.0 2.0+ 4.0+ 3.1+ 3.5+ 3.2+ 2.1+ 18.0+
7.0+
事例： 
p ~ p{color:#f00;}
							<p>p1</p> 
							<p>p2</p> 
							<h3>这是一个标题</h3>
							<p>p3</p> 
							<h3>这是一个标题</h3>
							<p>p4</p>
							<p>p5</p>

如果使用p ~ p{color:#f00;}，那么p2,p3,p4,p5将会变成红色；
